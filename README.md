# Lyssen

**Lyssen 是一个 macOS 菜单栏小工具，用来守护蓝牙耳机的播放音质。**

它解决的问题很具体：当 Mac 把蓝牙耳机同时当作“输出设备”和“麦克风输入设备”使用时，耳机经常会从高音质播放模式掉到通话模式，音乐、视频、游戏声音会突然变闷、变糊。Lyssen 会在后台自动把系统输入切回 Mac 内置麦克风，让蓝牙耳机继续保持高音质播放。

---

## 它解决什么问题

蓝牙耳机通常有两类工作模式：

| 模式 | 用途 | 听感 |
| --- | --- | --- |
| A2DP | 只做声音输出 | 立体声、高音质，适合听歌和看视频 |
| HFP / HSP | 同时做麦克风和声音输出 | 类似通话音质，码率低，声音发闷 |

当系统或会议软件把蓝牙耳机麦克风设为当前输入设备时，耳机往往会进入 HFP / HSP 通话模式。这个限制来自蓝牙协议本身，不是某一个播放器或 macOS 的单独问题。

Lyssen 的思路是：**蓝牙耳机继续负责输出，Mac 内置麦克风负责输入。**

这样你仍然可以使用麦克风，同时尽量避免蓝牙耳机播放音质被降级。

---

## 主要功能

- **蓝牙麦克风守护**  
  监听系统默认输入设备变化。一旦发现输入被切到蓝牙耳机，就自动切回内置麦克风。

- **默认后台运行**  
  启动后不占 Dock，只在菜单栏显示一个简洁的 `L`。点击 `L` 可以打开菜单。

- **轻量菜单栏菜单**  
  菜单里可以打开 Lyssen 窗口、暂停/开启守护、退出 Lyssen。

- **当前设备状态**  
  主界面显示当前输入设备、当前输出设备、累计守护次数。

- **可选拦截所有外置麦克风**  
  默认只拦截蓝牙输入。开启后，USB 麦克风、外置声卡等非内置输入也会被切回内置麦克风。

- **开机启动**  
  支持通过系统 `SMAppService` 注册开机启动。

- **中英双语**  
  右上角可以切换中文 / English，默认中文。

- **诊断信息复制**  
  可复制当前版本、开关状态、输入输出设备、守护次数等信息，方便排查问题。

- **快捷键**  
  打开窗口后，`Command + W` 关闭窗口但继续后台守护；`Command + Q` 退出整个应用。

## 工作原理

Lyssen 使用 macOS 原生 CoreAudio API 工作，没有音频录制、没有网络请求，也不修改音频内容。

核心流程如下：

1. 启动后定位 Mac 的内置麦克风。
2. 注册 CoreAudio 监听器，关注三类变化：
   - 默认输入设备变化
   - 默认输出设备变化
   - 系统音频设备列表变化
3. 当默认输入设备变化时，判断这个输入设备是否应该被拦截。
4. 如果当前输入是蓝牙设备，或你开启了“阻止外置麦克风”且当前输入不是内置设备，Lyssen 会调用 CoreAudio 把默认输入切回内置麦克风。
5. 界面同步刷新当前输入、当前输出和累计切换次数。

Lyssen 判断设备时主要依赖系统提供的传输类型和声道信息，而不是写死某个设备名称。因此不同 Mac 型号、不同蓝牙耳机名称通常都可以正常识别。

## 使用方式

1. 从 GitHub Release 下载最新版 `Lyssen.zip`。
2. 解压后双击 `Lyssen.app` 启动。
3. 右上角菜单栏会出现一个 `L`。
4. 点击 `L`，选择「打开 Lyssen」进入设置窗口。
5. 保持「蓝牙麦克风守护」开启即可。

关闭窗口不会退出 Lyssen，它会继续在后台守护。需要彻底退出时，点击菜单栏 `L`，选择「退出 Lyssen」。

## 设置说明

| 设置项 | 说明 |
| --- | --- |
| 开机启动 | 登录 macOS 后自动运行 Lyssen |
| 阻止外置麦克风 | 默认只拦截蓝牙；开启后会拦截所有非内置输入 |
| 启动时显示窗口 | 开启后启动 Lyssen 时自动显示主窗口 |
| 帮助与限制 | 查看使用限制说明 |
| 隐私说明 | 查看本地数据和网络使用说明 |
| 复制诊断信息 | 把当前状态复制到剪贴板 |

---

## 隐私

Lyssen 不联网，不上传数据，不录音，不读取音频内容。

它只读取系统音频设备列表、当前默认输入设备、当前默认输出设备，并在必要时修改系统默认输入设备。设置保存在本机 `UserDefaults` 中；诊断信息只会在你点击「复制诊断信息」时写入剪贴板。

---

## 已知限制

- 如果某个会议软件在自己的设置里强制选择蓝牙耳机麦克风，Lyssen 可能无法阻止该软件内部的选择。此时需要在该软件内手动把麦克风改成 Mac 内置麦克风。
- 如果菜单栏图标被系统或菜单栏管理工具隐藏，请检查右上角菜单栏收纳区，或重新启动 `Lyssen.app`。
- 开机启动依赖 macOS 的 ServiceManagement。若注册失败，可以在「系统设置」里手动把 Lyssen 加入登录项。

## 从源码重新编译

可以双击 `重新编译.command`，或在终端中执行：

```bash
swiftc "源码/main.swift" \
  -o "Lyssen.app/Contents/MacOS/Lyssen" \
  -framework AppKit \
  -framework SwiftUI \
  -framework CoreAudio \
  -framework ServiceManagement \
  -O

codesign --force --deep --sign - "Lyssen.app"
```
